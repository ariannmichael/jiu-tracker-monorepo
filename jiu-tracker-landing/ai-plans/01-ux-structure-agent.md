# UX / Structure Agent

## Mission

Turn the visual reference into a landing-page information architecture that maximizes clarity and conversion before any styling decisions get too deep.

## Inputs

- `ai-plans/00-master-brief.md`
- the provided reference image
- current app state: default Next.js starter

## Deliverables

1. Final section order for the landing page.
2. Wireframe-level content hierarchy for desktop and mobile.
3. CTA strategy:
   - primary CTA placements
   - secondary CTA placements
   - copy rules for buttons
4. Content density guidance per section.
5. Rules for what must be above the fold vs below the fold.

## Requirements

- Hero must communicate the app's purpose in one headline and one support paragraph.
- The first screen should include at least one primary CTA and one proof element.
- Feature sections should be grouped by user intent, not just by product internals.
- Mobile reading flow must remain clean when the right-side device visual stacks under the hero copy.
- The page must stay understandable even if a visitor does not inspect the screenshots closely.

## Recommended section blueprint

1. Hero
2. Three feature proof cards
3. Main feature grid
4. Performance / insights explanation
5. Download CTA section
6. Footer

## UX notes

- Use concise cards instead of dense paragraphs.
- Do not overload the hero with too many trust elements.
- Add a small label above the headline if it sharpens positioning, for example:
  - "Tracker para atletas de Jiu-Jitsu"
- Avoid carousels.
- Avoid hidden content critical to comprehension.

## Definition of done

- A visitor can understand the product, key benefits, and next action in under 30 seconds.
- The page structure supports both SEO scanability and visual polish.
